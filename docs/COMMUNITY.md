# Community Guide

## Communication

- Use issues for bugs and feature requests.
- Use pull requests for code/documentation changes.
- Keep discussions technical, reproducible, and actionable.

## Issue Types

- `bug`: incorrect behavior or regression
- `enhancement`: backward-compatible improvements
- `compatibility`: OOXML/PPTX compatibility gaps
- `performance`: throughput, memory, or startup concerns
- `security`: risky behavior (use private reporting path in `SECURITY.md`)

## Maintainer Expectations

- Triage incoming issues regularly.
- Request minimal reproductions for rendering mismatches.
- Label security-sensitive reports appropriately.
- Prefer evidence-driven decisions (tests, benchmarks, examples).

## Contributor Expectations

- Follow `CONTRIBUTING.md`.
- Follow `CODE_OF_CONDUCT.md`.
- Include tests for behavior changes whenever possible.
