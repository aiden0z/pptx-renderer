# v1.2.4 Release Notes

Release date: 2026-07-10

## Summary

v1.2.4 is a backward-compatible patch release focused on reliable browser integration,
optional runtime lifecycle management, and renderer hardening.

The release adds a standalone browser entry for no-bundler consumers while keeping
PDF.js optional. It also improves cleanup and cancellation for PDF fallback rendering,
uses modular ECharts registration, bounds untrusted CSS and SVG path parsing, and fixes
the horizontal scrollbar regression introduced by compact numeric no-wrap handling.

## Added

- Added the `@aiden0z/pptx-renderer/browser` ESM entry for direct browser use without an
  application bundler.
- Added real Chromium package tests for PPTX rendering, all supported ECharts series,
  and PDF.js 5/6 Worker compatibility.

## Changed

- ECharts now uses modular registration so the standalone browser bundle includes only
  the renderer's supported chart surface.
- Production builds now use a cross-platform Node build script.
- PDF.js remains an optional peer dependency and can be self-hosted by applications that
  need EMF-PDF fallback rendering.

## Fixed

- Fixed compact no-wrap text runs creating horizontal scrollbars in narrow text boxes.
- Fixed PDF.js loading-task cleanup and Worker termination across success, failure,
  timeout, and cancellation paths.
- Fixed late PDF fallback results mutating disposed slide DOM or repopulating revoked
  media caches.
- Replaced unsafe backtracking parsing of untrusted CSS and SVG path values with bounded
  parsing paths.

## Migration Notes

No API migration is required. Applications with a restrictive CSP that enable EMF-PDF
fallback must allow the configured PDF.js module source and `blob:` Workers.
